## Deno v2.6.10

[Deno v2.6.10](https://github.com/denoland/deno/releases/tag/v2.6.10)がリリースされています。

### `deno install --global --compile`

`deno install --global`で`--compile`オプションがサポートされています ([#32046](https://github.com/denoland/deno/pull/32046))

指定されたエントリーポイントをインストールする際に`deno compile`を適用して実行可能ファイルを作成してくれます。

```shell
$ deno install --global --compile -ER npm:cowsay

$ cowsay hi
 ____
< hi >
 ----
        \   ^__^
         \  (oo)\_______
            (__)\       )\/\
                ||----w |
                ||     ||
```

### `deno install`

`deno install`/`deno add`/`deno remove`の実行時に、不要なnpmパッケージを`node_modules`から削除してくれるよう挙動が改善されています ([#32058](https://github.com/denoland/deno/pull/32058))

### `SSLKEYLOGFILE`

[`SSLKEYLOGFILE`](https://github.com/curl/curl/blob/8494012196474ee0541940140344cd24ed340301/docs/cmdline-opts/_ENVIRONMENT.md#sslkeylogfile-path)環境変数がサポートされています ([#31867](https://github.com/denoland/deno/pull/31867))

Wiresharkなどでトラフィックを解析するために、指定されたファイルに対してTLSのセッションキーを書き込んでくれます。

### `deno publish`

`COPYING`ファイルが[ライセンスファイル]({{< ref "articles/deno/v1.46.md" >}})として認識されるよう改善されています ([#32133](https://github.com/denoland/deno/pull/32133))

### `node:worker_threads`

`Worker.stdin`がサポートされています ([#32165](https://github.com/denoland/deno/pull/32165))。`Worker`スレッド内からは`process.stdin`経由で`Worker.stdin`に書き込まれた内容を読み込めます。

同様に`Worker`スレッド内から`process.stdout`または`process.stderr`に書き込まれた内容が`Worker.stdout`または`Worker.stderr`の経由で読み込めるようにする改善が実施されています ([#32160](https://github.com/denoland/deno/pull/32160))

また、`Worker`で`execArgv`オプションのサポートが追加されています ([#32145](https://github.com/denoland/deno/pull/32145))

### `node:fs`

`openSync()`で`O_RDONLY | O_CREAT`の組み合わせが動作するよう改善されています ([#32152](https://github.com/denoland/deno/pull/32152))

また、`opendir()`などが返却する`Dirent`クラスにおいて`parentPath`プロパティーが設定されるよう改善されています ([#32178](https://github.com/denoland/deno/pull/32178))

### `node:buffer`

`File`クラスが`export`されています ([#32182](https://github.com/denoland/deno/pull/32182))

### `AbortSignal.any()`

`AbortSignal.any()`で作成した`AbortSignal`が意図せずGCによって回収されてしまうことがある問題が修正されています ([#32118](https://github.com/denoland/deno/pull/32118))

### `Deno.test()`

`Deno.test()`で定義したテストケース中でネストして`Deno.test()`が呼ばれた際はエラーが発生するよう挙動が改善されています ([#32184](https://github.com/denoland/deno/pull/32184))

テストケースをネストさせたい場合は`Deno.TestContext#step()`の使用が推奨されます。

## DenoとVultrのパートナーシップについて

Denoが[Vultr Cloud Alliance](https://www.vultr.com/cloudalliance/)へ参加したことが発表されました。

- [Modern JavaScript and TypeScript Applications Built with Deno on Vultr](https://blogs.vultr.com/deno-cloud-alliance)

元々、Deno Deployの内部で[Vultr](https://www.vultr.com/)が使用されていたようで、その関係でパートナーシップが結ばれたようです。

また、Vultrの以下の公式ドキュメントにて[Vultr Serverless Inference](https://www.vultr.com/products/cloud-inference/)と[Deno Sandbox]({{< ref "articles/2026/01/25.md" >}})の連携例が紹介されています:

- [How to Utilize Deno Sandboxes With Vultr](https://docs.vultr.com/how-to-utilize-deno-sandboxes-with-vultr)

## Hono v4.12.0

[Hono v4.12.0](https://github.com/honojs/hono/releases/tag/v4.12.0)がリリースされています。

### `hono/ssg`

`redirectPlugin`が追加されています ([#4599](https://github.com/honojs/hono/pull/4599))。301などのステータスコードを返却したページが検出された場合、`Location`ヘッダーの設定値に基づいて[`http-equiv="refresh"`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/meta/http-equiv)が設定された静的なHTMLページを生成してくれます。

### `hono/client`

`ApplyGlobalResponse`型が追加されています ([#4556](https://github.com/honojs/hono/pull/4556))。主にエラーレスポンスに関する型をクライアントが推論できるようにすることを目的に追加されたようです。

クライアントに`$path()`メソッドが追加されています ([#4636](https://github.com/honojs/hono/pull/4636))。`$url()`と異なり、パスを文字列で返却してくれます。

### `hono/basic-auth`

`onAuthSuccess`コールバックがサポートされています ([#4645](https://github.com/honojs/hono/pull/4645))。認証成功時に`Context`及びユーザー名を引数にコールバックが呼ばれます。

### `hono/trailing-slash`

`trimTrailingSlash()`及び`appendTrailingSlash()`に`alwaysRedirect`オプションが追加されています ([#4658](https://github.com/honojs/hono/pull/4658))。

例えば、`trimTrailingSlash()`で`alwaysRedirect: true`を設定した場合、末尾が`/`で終わる`GET`または`HEAD`リクエストを受信した場合、常に末尾の`/`を取り除いたパスへリダイレクトするようです。

### `hono/language`

[RFC 4647](https://datatracker.ietf.org/doc/html/rfc4647)が実装されています ([#4717](https://github.com/honojs/hono/pull/4717))。これにより、例えば`ja-JP`が`ja`にマッチしてくれるよう挙動が改善されるようです。

### `hono/router/trie-router`

`TrieRouter`の内部実装の改善により、パフォーマンスが1.5〜2倍近くまで高速化されたようです ([#4724](https://github.com/honojs/hono/pull/4724))
