## Deno v2.6.5

[Deno v2.6.5](https://github.com/denoland/deno/releases/tag/v2.6.5)がリリースされています。

### Web API

#### `createImageBitmap()`

`createImageBitmap()`への入力としてgifやwebp形式がサポートされています ([#31402](https://github.com/denoland/deno/pull/31402))

#### `Performance`

[`performance.clearResourceTimings()`](https://developer.mozilla.org/en-US/docs/Web/API/Performance/clearResourceTimings)及び[`performance.setResourceTimingBufferSize()`](https://developer.mozilla.org/en-US/docs/Web/API/Performance/setResourceTimingBufferSize)が実装されています ([#31603](https://github.com/denoland/deno/pull/31603))

#### WebGPU

`GPUCompilationMessage`の`offset`及びlengthプロパティーの型定義が追加されています ([#31780](https://github.com/denoland/deno/pull/31780))

### `deno audit`

`deno audit`コマンドで何らかの脆弱性が検出された際に常に終了コードとして1が返却される問題が修正されています。`--level`で指定された以上の脆弱性が存在しない場合は終了コードとして1を返さないよう修正されています ([#31721](https://github.com/denoland/deno/pull/31721))

### `deno compile`

`deno compile`で`--include`オプションによって埋め込まれたファイルを`fetch()` API経由で読み込む※ことが出来なかった問題が修正されています ([#31814](https://github.com/denoland/deno/pull/31814))

※ Denoでは `fetch()` APIに `file:` 形式のURLなどを指定することで、ローカルのファイルを読み込むことが出来ます。

### `node:fs/promises`

`FileHandle#readableWebStream()`が実装されています ([#31745](https://github.com/denoland/deno/pull/31745))

### `node:tls`

`setDefaultCACertificates()`が実装されています ([#31522](https://github.com/denoland/deno/pull/31522))

### `node:sqlite`

`DatabaseSync`がGCされると、それによって作成された`StatementSync`も無効化されてしまう問題が修正されています ([#31737](https://github.com/denoland/deno/pull/31737))

また、問い合わせ結果の走査中に意図せず`StatementSync`がGCされないように修正されています ([#31770](https://github.com/denoland/deno/pull/31770))

### Deno API

`Deno.errors`配下の各種エラーのコンストラクターに`options`引数を渡した際に、それが無視されてしまう問題が修正されています ([#31510](https://github.com/denoland/deno/pull/31510))

### CLI

Top-level await を使用したモジュールが複数回同時に読み込まれると、エラーが発生する問題が修正されています ([#31670](https://github.com/denoland/deno/pull/31670))

## LogTape v2

ロギングライブラリのLogTapeでv2のリリースが行われています。

- [リリースノート](https://github.com/dahlia/logtape/discussions/133)
- [CHANGES.md](https://github.com/dahlia/logtape/blob/83444eaf5945776cac565e7e40406fe21c24b47c/CHANGES.md)

`lazy()`というAPIが追加されています。このAPIは`Logger#with()`との併用が想定されており、ロギングが実行される度にコンテキストの動的な計算を行うことができます。

また、下記メソッドの第1引数に直接`Error`オブジェクトを渡せるようにする改善が実施されています。

- `Logger#error()`
- `Logger#warn()`
- `Logger#fatal()`

[@logtape/config](https://jsr.io/@logtape/config)というパッケージが追加されています。このパッケージはJSONやYAMLなどからLogTapeを設定できるようにすることなどが目的とされているようです。

[@logtape/file](https://jsr.io/@logtape/file)パッケージで`getTimeRotatingFileSink()`が追加されています。日時に基づいたログのローテションがサポートされています。
