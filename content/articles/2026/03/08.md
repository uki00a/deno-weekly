## Deno v2.7.2/v2.7.3/v2.7.4

以下のバージョンのリリースが実施されています。

- [Deno v2.7.2](https://github.com/denoland/deno/releases/tag/v2.7.2)
- [Deno v2.7.3](https://github.com/denoland/deno/releases/tag/v2.7.3)
- [Deno v2.7.4](https://github.com/denoland/deno/releases/tag/v2.7.4)

### `deno install`

#### パフォーマンス改善

並列性の改善により`deno install`によるnpmパッケージのインストール速度が改善されています ([#32416](https://github.com/denoland/deno/pull/32416))

[パッケージによっては4倍以上の改善がなされる](https://x.com/deno_land/status/2028869557608235456)ケースもあるようです。

[https://x.com/rough__sea/status/2029531069435830602](https://x.com/rough__sea/status/2029531069435830602)によると、この改善により各種パッケージマネージャー内において Bun に次ぐ速度が達成されているようです。

#### `peerDependencies`

`peerDependencies`の解決が改善されており、あるパッケージが複数のパッケージから`peerDependencies`として依存されている場合、意図せずそのパッケージの複数のバージョンがインストールされてしまうことがある問題への改善が実施されています ([#32358](https://github.com/denoland/deno/pull/32358))

### `deno compile --self-extracting`

[*extraction directory*]({{< ref "articles/deno/v2.7.md" >}}) の名前が変更されています ([#32329](https://github.com/denoland/deno/pull/32329))

- **変更前**: `<実行可能ファイル>.fs/<ハッシュ>`
- **変更後**: `.<実行可能ファイル>/<ハッシュ>` (`.fs`が削除され、隠しファイルとして扱われるよう`.`がプレフィックスとして付与されています)

### `deno jupyter`

以下の改善が実施されています ([#32359](https://github.com/denoland/deno/pull/32359))

- Denoカーネルで[`interrupt_request`](https://github.com/jupyter/jupyter_client/blob/891e4e44dbf04bf109cb2fd9334e9e38f98a23ca/docs/messaging.rst#kernel-interrupt)メッセージがサポートされています
- [`shutdown_request`](https://github.com/jupyter/jupyter_client/blob/891e4e44dbf04bf109cb2fd9334e9e38f98a23ca/docs/messaging.rst#kernel-shutdown)に対して、Denoカーネルが`shutdown_reply`を返却するように改善されています

### `node:crypto`

下記 API が実装されています:

- `X509Certificate` の下記API ([#32270](https://github.com/denoland/deno/pull/32270))
    - `checkIP()`
    - `checkIssued()`
    - `checkPrivateKey()`
    - `infoAccess`
    - `raw`
    - `subjectAltName`
    - `toLegacyObject()`
    - `toString()`
    - `verify()`
- `KeyObject#equals()` ([#32409](https://github.com/denoland/deno/pull/32409))

また、下記改善も実施されています:

- ECDSA鍵の生成/署名などにおけるsecp256k1曲線 (`namedCurve: "secp256k1"`) のサポート ([#32390](https://github.com/denoland/deno/pull/32390))
- `publicEncrypt()`, `privateDecrypt()`及び`privateEncrypt()`でDER形式の鍵がサポート ([#32291](https://github.com/denoland/deno/pull/32291))
- `Sign#sign()`及び`Verify#verify()`で`padding: constants.RSA_PKCS1_PSS_PADDING`の指定がサポート ([#32269](https://github.com/denoland/deno/pull/32269))
- EC鍵のSEC1形式のimport及びexportがサポート ([#32401](https://github.com/denoland/deno/pull/32401))

### `node:dgram`

下記 API が実装されています ([#32381](https://github.com/denoland/deno/pull/32381))

- `Socket#addSourceSpecificMembership()`
- `Socket#dropSourceSpecificMembership()`
- `Socket#setMulticastInterface()`
- `Socket#setTTL()`

また、Node.js に合わせて`Socket#addMembership()`や`Socket#setBroadcast()`などの各種設定操作を同期的に動作させる改善も実施されています。

### `node:http`

`request()`の`socketPath`オプションでWindowsの名前付きパイプの指定がサポートされています ([#32414](https://github.com/denoland/deno/pull/32414))

[dockerode](https://github.com/apocas/dockerode)などを Windows でサポートすることが目的のようです。

### `node:assert`

`node:assert`の互換性が改善され、Node.jsにおける`node:assert`のテストスイートすべてパスしたようです ([#32434](https://github.com/denoland/deno/pull/32434))

### `node:worker_threads`

`eval: true`を指定して作成された`Worker`内で`require()`や`__dirname`などのCommonJS向けAPIが利用できるよう改善されています ([#32266](https://github.com/denoland/deno/pull/32266))

### `node:fs/promises`

`watch()`が返却したオブジェクトに適切に`AsyncIterator`を実装させる改善や、`signal`引数で指定されたシグナルがすでに`abort`されていた際の取り扱いが改善されています ([#32378](https://github.com/denoland/deno/pull/32378))

### `node:process`

Windowsにおいて、`Deno.addSignalListener()`及び`process.on()`で`SIGWINCH`がサポートされています ([#32343](https://github.com/denoland/deno/pull/32343))

また、`umask()`が実装されています ([#32385](https://github.com/denoland/deno/pull/32385))

### `node:buffer`

`resolveObjectURL()`が実装されています ([#32382](https://github.com/denoland/deno/pull/32382))

### `node:vm`

`createContext()`で`constants.DONT_CONTEXTIFY`の指定がサポートされています ([#32337](https://github.com/denoland/deno/pull/32337))

### `require(esm)`

例外を投げるESMファイルを`require()`した際に、その例外をキャッチ出来ない問題が修正されています ([#32347](https://github.com/denoland/deno/pull/32347))

### `NPM_CONFIG_REGISTRY`

[`NPM_CONFIG_REGISTRY`]({{< ref "articles/deno/v1.29.md" >}})と`.npmrc`が併用された際に、`NPM_CONFIG_REGISTRY`の方が優先されるように修正されています ([#32394](https://github.com/denoland/deno/pull/32394))

### `console.table()`

`console.table()`に`Iterator`オブジェクトが渡された際に、空のテーブルではなく各要素の内容が適切に出力されるように改善されています ([#32379](https://github.com/denoland/deno/pull/32379))
