## Deno v2.7.8 & Deno v2.7.9

[Deno v2.7.8](https://github.com/denoland/deno/releases/tag/v2.7.8)と[Deno v2.7.9](https://github.com/denoland/deno/releases/tag/v2.7.9)がリリースされています。

### OpenTelemetry

#### `OTEL_EXPORTER_OTLP_PROTOCOL=console`

`OTEL_EXPORTER_OTLP_PROTOCOL=console`がサポートされています ([#32717](https://github.com/denoland/deno/pull/32717))。これが指定されると、シグナルがOpenTelemetryバックエンドではなく標準エラー出力に書き込まれます。

使用例として、[`DENO_AUDIT_PERMISSIONS=otel`]({{< ref "articles/2026/03/15.md" >}})と併用することで、アプリケーションが要求したパーミッションをログに出力することができます:

```shell
$ cat main.js
console.info(Deno.env.get("FOO"));



$ OTEL_DENO=true \
    DENO_AUDIT_PERMISSIONS=otel \
    OTEL_EXPORTER_OTLP_PROTOCOL=console \
    deno run -E=FOO main.js

LOG [INFO] 2026-03-29T07:33:36.045Z "env: FOO"
  scope: deno@2.7.9
  deno.permission.type: "env"
  deno.permission.value: "FOO"
  deno.permission.stack: ""
LOG [INFO] 2026-03-29T07:33:36.046Z "foo
"
  scope: deno@2.7.9
  log.iostream: "stdout"
```

#### `http.route`

`Deno.serve()`のハンドラー内から`trace.getActiveSpan()`で取得した`Span`において`http.route`属性の設定がサポートされています ([#32720](https://github.com/denoland/deno/pull/32720))

### `deno install -g`

#### jsrパッケージのインストールが改善

`deno install -g`でjsrパッケージをインストールする際に、そのパッケージが提供する`deno.lock`をベースに依存解決が行われるように挙動が改善されています ([#32490](https://github.com/denoland/deno/pull/32490))

#### 複数の`bin`を定義するパッケージのサポート

`package.json`の`bin`エントリーで複数の実行可能ファイルを指定しているnpmパッケージについて、最初の実行可能ファイルに対してしか shim が作成されない問題が修正されています ([#32607](https://github.com/denoland/deno/pull/32607))

### `deno approve-scripts` - Windowsにおける振る舞いの改善

Windowsにおける[`deno approve-scripts`]({{< ref "articles/deno/v2.6.md" >}})や[`deno update --interactive`]({{< ref "articles/deno/v2.2.md" >}})で表示されるインタラクティブなUIにおいて、Ctrl+Cや矢印キーによる操作、Enterなどがサポートされています ([#32978](https://github.com/denoland/deno/pull/32978))

### `deno update` - `--frozen=false`のサポート

`deno update`コマンドで[`--frozen=false`]({{< ref "articles/deno/v1.45.md" >}})がサポートされています ([#32975](https://github.com/denoland/deno/pull/32975))

普段の運用において`deno.json`で[`lock.frozen: true`]({{< ref "articles/deno/v1.46.md" >}})を設定しているようなケースにおいて役立ちそうです。

### `deno fmt --unstable-component`

[`deno fmt --unstable-component`]({{< ref "articles/deno/v1.46.md" >}})の安定性の改善が実施されています ([#29763](https://github.com/denoland/deno/pull/29763))

また、`deno.json`において`deno fmt --unstable-component`向けの下記設定項目が追加されています:

- [`fmt.vueComponentCase`](https://github.com/g-plane/markup_fmt/blob/40a8bf949264e26928ce9b0da7e27bea868fb25d/docs/src/config/vue-component-case.md)
- [`fmt.angularNextControlFlowSameLine`](https://github.com/g-plane/markup_fmt/blob/40a8bf949264e26928ce9b0da7e27bea868fb25d/docs/src/config/angular-next-control-flow-same-line.md)

### `--env-file`

[`--env-file`]({{< ref "articles/deno/v1.46.md" >}})で`.env`を読み込む際にネストされたクォートなどがうまく取り扱われない問題が修正されています ([#32930](https://github.com/denoland/deno/pull/32930))

例えば、以下のような`.env`があったとします。

```shell
FOO="foob"ar"
```

この場合、以下のように出力されます:

```javascript
console.info(Deno.env.get("FOO"));
// v2.7.7: "foob"
// v2.7.9: "foob\"ar"
```

### `--inspect`

下記メソッドが実装されています ([#32714](https://github.com/denoland/deno/pull/32714))

- `NodeRuntime.enable`
- `NodeRuntime.disable`
- `Runtime.runIfWaitingForDebugger`

### Web API

#### `AbortSignal.any()`

`AbortSignal.any()`でメモリリークが起きることがある問題が修正されています ([#32916](https://github.com/denoland/deno/pull/32916))

`AbortSignal.any()`の引数に長期間維持され続けている`AbortSignal`が繰り返し渡されているようなケースにおいて再現することがあったようです。

#### Web Crypto API

`SubtleCrypto#sign()`および`SubtleCrypto#verify()`で`namedCurve: "P-521"`を指定して作成された鍵がサポートされています ([#32602](https://github.com/denoland/deno/pull/32602))

#### `EventTarget#removeEventListener()`

`EventTarget#removeEventListener()`の第3引数に`null`が与えられた際にエラーが発生する問題が修正されています ([#32605](https://github.com/denoland/deno/pull/32605))

### Node.js互換性の改善

#### `node:sqlite`

下記の改善が実施されています ([#32756](https://github.com/denoland/deno/pull/32756))

- `DatabaseSync#prepare()`で`options`引数がサポート
- `defensive`オプションのデフォルト値が`false`から`true`へ変更
- `StatementSync`の `all()`/`get()`/`iterate()`/`run()`が呼ばれた際に、 該当の`StatementSync`から作成されていたアクティブな`Iterator`が無効化されるように挙動が修正

#### `node:process`

下記APIが実装されています:

- `process.threadCpuUsage()` ([#32829](https://github.com/denoland/deno/pull/32829))
- `process.getgroups()` ([#32772](https://github.com/denoland/deno/pull/32772))
- `process.setUncaughtExceptionCaptureCallback()`/`process.hasUncaughtExceptionCaptureCallback()` ([#32897](https://github.com/denoland/deno/pull/32897))

#### `node:fs`

`opendir()`などが返却する`Dir`クラスにおいて`[Symbol.dispose]`及び`[Symbol.asyncDispose]`が実装されています ([#33000](https://github.com/denoland/deno/pull/33000))

#### `node:child_process`

`spawn()`で`uid`及び`gid`オプションがサポートされています ([#32772](https://github.com/denoland/deno/pull/32772))

#### `node:events`

`on()`で`highWaterMark`及び`lowWaterMark`オプションのサポートが実施されています ([#32826](https://github.com/denoland/deno/pull/32826))

#### `node:buffer`

`Buffer.concat()`に4GBより大きい`Buffer`が与えられると、データの一部が意図せず切り取られてしまう問題が修正されています ([#32913](https://github.com/denoland/deno/pull/32913))

#### `node:tty`

`ReadStream#destroy()`や`WriteStream#destroy()`を呼んでもfdが解放されず、リークが発生してしまう問題が修正されています ([#32958](https://github.com/denoland/deno/pull/32958))

#### `node:http`

`request()`で`keepAlive: true`が指定された`Agent`を使用している場合、`close`よりも先に`free`が発火してしまうことがあった問題が修正されています ([#32811](https://github.com/denoland/deno/pull/32811))

#### `node:https`

`request()`に渡した`createConnection()`オプションが`tls.connect()`によって事前に作成された`TLSSocket`を返却した場合、メモリリークが発生してしまう問題が修正されています ([#32961](https://github.com/denoland/deno/pull/32961))

#### `node:domain`

タイマーのコールバックなどにおいて`process.domain`が失われてしまう問題の修正や`Domain`クラスの各種メソッドの互換性の改善が実施されています ([#32897](https://github.com/denoland/deno/pull/32897))

#### `package.json`

`package.json`の`dependencies`にで`file:`または`link:`を依存として持つnpmパッケージがあるとエラーが発生してしまう問題が修正されています ([#32876](https://github.com/denoland/deno/pull/32876))

#### `"nodeModulesDir": "manual"`

`"nodeModulesDir": "manual"`が有効化されている場合、ユーザーが指定したバージョンではなく、キャッシュされているバージョンの中で最も高いバージョンのパッケージが使用されてしまうことがある問題が修正されています ([#32977](https://github.com/denoland/deno/pull/32977))

### パフォーマンス改善

`btoa()`や`atob()`, `Buffer.toString("base64")`などの各種Base64関連のエンコード/デコード処理が最適化されています ([#32743](https://github.com/denoland/deno/pull/32743))
