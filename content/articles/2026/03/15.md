# Deno v2.7.5

[Deno v2.7.5](https://github.com/denoland/deno/releases/tag/v2.7.5)がリリースされています。

### `--cpu-prof`

`--cpu-prof`, `--cpu-prof-dir`, `--cpu-prof-name`, `--cpu-prof-interval`, 及び`--cpu-prof-md`オプションが追加されています ([#31909](https://github.com/denoland/deno/pull/31909))

あわせて公式ドキュメントの追加も実施されています (https://github.com/denoland/docs/commit/71602309d18c515c560e54c0ffe3be709156610a)

`--cpu-prof`オプションを指定すると、CPU プロファイラーが有効化され、プロセス終了時にデフォルトでカレントディレクトリに CPU プロファイルが保存されます (デフォルトで `CPU.<timestamp>.<pid>.cpuprofile` という名前で保存されます。`--cpu-prof-dir`オプションで出力先ディレクトリ、`--cpu-prof-name`オプションによってファイル名をカスタマイズできます)

```shell
$ deno run --cpu-prof --cpu-prof-name=cli.cpuprofile -R src/cli.ts

$ file cli.cpuprofile 
cli.cpuprofile: JSON data
```

また、`--cpu-prof-md` オプションを指定すると、プロファイル結果を Markdown 形式で出力してくれます。

```shell
$ deno run --cpu-prof-md --cpu-prof-name=cli.cpuprofile -R src/cli.ts

$ cat cli.md
# CPU Profile

| Duration | Samples | Interval | Functions |
| --- | --- | --- | --- |
| 414.49ms | 29 | 1000us | 90 |

**Top 10:** `scanNext` 20.0%, `firstString` 10.0%, `cache` 10.0%, `op_require_real_path` 10.0%, `compileFunction` 10.0%, `normalizeString` 10.0%, `findNodeAtLocation` 10.0%, `onLiteralValue` 10.0%, `lintText` 10.0%

## Hot Functions

| Self% | Self Time | Samples | Function | Location |
| ---: | ---: | ---: | --- | --- |
| 20.0% | 2.00ms | 2 | `scanNext` | impl/parser.js:412 |

  ... 省略 ...
```

`--cpu-prof-interval`オプションはサンプリング間隔を調整したい場合に利用できます (デフォルト値は 1000 です)

### OpenTelemetry

[`DENO_AUDIT_PERMISSIONS`]({{< ref "articles/deno/v2.5.md" >}})環境変数に`otel`の指定がサポートされています ([#32501](https://github.com/denoland/deno/pull/32501))

`DENO_AUDIT_PERMISSIONS=otel`が指定されていると、アプリケーションから要求されたパーミッションをローカルファイルへは書き込まず、代わりに *Log* として OpenTelemetry バックエンドへ送信されます。

### `deno coverage`

`deno coverage`コマンドで関数のカバレッジがサポートされています  ([#32507](https://github.com/denoland/deno/pull/32507))

`deno coverage`が表示するテーブルに`Function %`列が追加されています:

```shell
| File           | Branch % | Function % | Line % |
| -------------- | -------- | ---------- | ------ |
| permissions.ts |     94.3 |      100.0 |   97.0 |
| All files      |     94.3 |      100.0 |   97.0 |
```

また、[HTMLレポート]({{< ref "articles/deno/v1.39.md" >}})にも同様の出力が追加されています。

## `deno doc npm:<package>`

 `deno doc`コマンドでnpmパッケージ (`npm:<package>`) がサポートされています ([#32435](https://github.com/denoland/deno/pull/32435))

該当パッケージが提供する型定義などを元にドキュメントを表示してくれるようです:

```shell
$ deno doc npm:lines-and-columns
Defined in file:///path/to/npm/registry.npmjs.org/lines-and-columns/2.0.4/build/index.d.ts:5:1

class LinesAndColumns

  constructor(string: string)
  locationForIndex(index: number): SourceLocation | null
  indexForLocation(location: SourceLocation): number | null

Defined in file:///path/to/npm/registry.npmjs.org/lines-and-columns/2.0.4/build/index.d.ts:1:1

interface SourceLocation

  line: number
  column: number
```

### `--env`

`.env`において環境変数の置き換えがサポートされています ([#32495](https://github.com/denoland/deno/pull/32495))

例えば、以下のような`.env`があったとします:

```shell
KEY=foo
KEY2=${KEY}_bar
```

この`.env`を`--env`によって読み込んだ場合、``以下のように出力されます:

```javascript
console.info(Deno.env.get("KEY"));  // => foo
console.info(Deno.env.get("KEY2")); // => foo_bar
```

### Node.js 互換性の改善

#### `node:fs`

`cp()`の内部実装の多くがRustで書き直され、パフォーマンスと互換性が改善されています ([#32580](https://github.com/denoland/deno/pull/32580))

Linux環境においては3倍近くまでパフォーマンスが改善されるケースもあるようです。

#### `node:http2`

[denoland/libnghttp2](https://github.com/denoland/libnghttp2)をベースに`node:http2`が再実装され、互換性が大幅に改善されています ([#32418](https://github.com/denoland/deno/pull/32418))

この変更により、Node.jsにおける`node:http2`の134個のテストスイートのうち130個がパスされるまで互換性が改善されているようです。

#### `node:crypto`

下記 API が実装されています:

- `ECDH.convertKey()` ([#32532](https://github.com/denoland/deno/pull/32532))
- `KeyObject#toCryptoKey()`及び`KeyObject.from()` ([#32523](https://github.com/denoland/deno/pull/32523))
    - この改善に合わせて`SubtleCrypto#exportKey()`及び`SubtleCrypto#importKey(`)で `ChaCha20-Poly1305` がサポートされています
- `createCipheriv()`/`createDecipheriv()`での`des-ede3-cbc`のサポート ([#32519](https://github.com/denoland/deno/pull/32519))
- `getCipherInfo()` - `nid`及び`blockSize`プロパティーが返却されるように改善 ([#32522](https://github.com/denoland/deno/pull/32522))

#### `node:dns`

下記の改善が実施されています ([#32536](https://github.com/denoland/deno/pull/32536))

- `setDefaultResultOrder()`で`ipv6first`がサポートされています
- `lookup()`の`order`オプションがサポートされています
- カスタムサーバーが未設定の場合、`Resolver#getServers()` が`/etc/resolv.conf`を参照するように改善されています

#### `node:process`

`nextTick()`で登録したコールバックがNode.jsと同様のタイミングで実行されるように改善されています ([#32466](https://github.com/denoland/deno/pull/32466))

元々、`nextTick()`で登録したコールバックが`Promise`の`.then()`で登録したコールバックよりも後に実行されてしまう問題があったようです。

#### `node:readline`

下記キーバインディングが実装されています ([#32538](https://github.com/denoland/deno/pull/32538))

- *Ctrl*+*-* (Undo)
- *Ctrl*+*^* (Redo)
- *Ctrl*+*Y* (Yank)
- *Meta*+*Y*

#### `node:worker_threads`

`resourceLimits`が実装されています ([#32430](https://github.com/denoland/deno/pull/32430))

#### `node:v8`

下記APIが実装されています ([#32483](https://github.com/denoland/deno/pull/32483))

- `getHeapStatistics()`
- `getHeapCodeStatistics()`
- `getHeapSnapshot()`
- `writeHeapSnapshot()` (利用には`write`パーミッションが必要です)

#### `node:dgram`

`Socket#[Symbol.asyncDispose]()` 及び `createSocket()`の`ipv6Only`オプションが実装されています ([#32520](https://github.com/denoland/deno/pull/32520))

#### Node-API

下記APIが実装されています:

- `node_api_create_property_key_latin1()`及び`node_api_create_property_key_utf8()` ([#32559](https://github.com/denoland/deno/pull/32559))
- `node_api_create_object_with_properties()` ([#31443](https://github.com/denoland/deno/pull/31443))

#### `.npmrc`

`email`による認証がサポートされています ([#32616](https://github.com/denoland/deno/pull/32616))
