## Deno v2.7.6 & Deno v2.7.7

[Deno v2.7.6](https://github.com/denoland/deno/releases/tag/v2.7.6)と[Deno v2.7.7](https://github.com/denoland/deno/releases/tag/v2.7.7)がリリースされています。

### `--cpu-prof`

`--cpu-prof-flamegraph`オプションが追加されています ([#32572](https://github.com/denoland/deno/pull/32572), [#32716](https://github.com/denoland/deno/pull/32716))

フレームグラフをSVG形式で出力してくれます:

```shell
$ deno run --cpu-prof-flamegraph --cpu-prof-name=cli.cpuprofile --cpu-prof-dir=cpu-profile -R src/cli.ts

$ open cpu-profile/cli.svg
```

`--cpu-prof-flamegraph`オプションの公式ドキュメントも追加されています ([denoland/docs#2975](https://github.com/denoland/docs/pull/2975))

また、[`--cpu-prof`]({{< ref "articles/2026/03/15.md" >}})オプション関連の出力に対してソースマップが適用されるよう改善されています ([#32634](https://github.com/denoland/deno/pull/32634))

### `deno eval`

`--ext`オプションを指定しなくても、与えられたソースコードがCJSかESMかを自動的に判定してくれるように改善されています ([#32472](https://github.com/denoland/deno/pull/32472))

### シグナル

`--watch`オプション使用時にファイルの変更もしくはCtrl+Cの入力が検出された際に、`SIGTERM`シグナル向けのリスナーが呼ばれるように挙動が改善されています ([#32564](https://github.com/denoland/deno/pull/32564))

また、Windowsで下記改善が実施されています ([#32689](https://github.com/denoland/deno/pull/32689))

- `Deno.addSignalListener()`で`SIGQUIT`と`SIGTERM`がサポート
- `Deno.kill()`で`SIGQUIT`/`SIGKILL`/`SIGTERM`/`SIGABRT`がサポート

### OpenTelemetry

`console.log()`や`console.error()`などの第1引数に`Error`オブジェクトが渡された場合、[Semantic Conventions](https://github.com/open-telemetry/semantic-conventions/blob/af46181baec13bd5fe5552b3ec8181743047ffae/docs/exceptions/exceptions-logs.md#attributes)に従って _Log_ レコードに下記の属性が追加されるように改善されています ([#32726](https://github.com/denoland/deno/pull/32726))

- `exception.type`
- `exception.message`
- `exception.stacktrace`

また、_Log_ レコードに[`log.iostream`](https://github.com/open-telemetry/semantic-conventions/blob/1b8c3ba4ef1bf97f79553fdfe558fe4c2a70a0e5/docs/registry/attributes/log.md#general-log-attributes)属性を設定する改善も実施されています ([#32723](https://github.com/denoland/deno/pull/32723))。`console.error()`または`console.warn()`によるログの出力の場合は`stderr`, それ以外の場合は`stdout`が設定されます。

また、_Attributes_ への配列の指定がサポートされています ([#32748](https://github.com/denoland/deno/pull/32748))

### Node.js互換性の改善

#### `deno install`

書き込み権限が付与されていない実行可能ファイル (`bin`) を配布するnpmパッケージのインストールに失敗してしまう問題が修正されています ([#32632](https://github.com/denoland/deno/pull/32632))

#### `node:util`

下記APIが実装されています ([#32793](https://github.com/denoland/deno/pull/32793)):

- `MIMEType`
- `MIMEParams`
- `convertProcessSignalToExitCode()`

#### `node:child_process`

`ChildProcess#[Symbol.dispose]()`が実装されています ([#32793](https://github.com/denoland/deno/pull/32793))

また、親プロセス終了時に`unref()`されたプロセスも終了してしまう問題が修正されています ([#32563](https://github.com/denoland/deno/pull/32563))

#### `node:process`

`process.title`の取り扱いが以下のように改善されています ([#32201](https://github.com/denoland/deno/pull/32201))

- `process.title`のデフォルト値が`"deno"`から`process.execPath`に変更されています
- `process.title`の上書きが適切に機能するように改善されています
- `NODE_OPTIONS`に`--title`が含まれる場合、その値が`process.title`に自動設定されるよう挙動が改善されています

また、`process.once()`で登録したリスナーが`process.removeListener()`で削除できない問題が修正されています ([#32606](https://github.com/denoland/deno/pull/32606))

#### `node:crypto`

以下の改善が実施されています:

- `publicDecrypt()`が実装されています ([#32690](https://github.com/denoland/deno/pull/32690))
- `publicEncrypt()`及び`privateDecrypt()`で`KeyObject`の指定がサポートされています ([#32690](https://github.com/denoland/deno/pull/32690))
- `KeyObject#export()`で`format === "pem"`形式でのキーのエクスポートがサポートされています ([#32703](https://github.com/denoland/deno/pull/32703))
- `CryptoKey`オブジェクトに対する`structuredClone()`のサポートが実施されています ([#32674](https://github.com/denoland/deno/pull/32674))
- `X509Certificate` - `signatureAlgorithm/signatureAlgorithmOid`プロパティーの実装 及び `structuredClone()`のサポートが行われています ([#32671](https://github.com/denoland/deno/pull/32671))
- `generatePrime()`で`add`/`rem`/`safe`オプションがサポートされています ([#32618](https://github.com/denoland/deno/pull/32618))

#### `node:fs`

`createReadStream()`及び`createWriteStream()`の`fd`オプションで`FileHandle`クラスの指定がサポートされています ([#32770](https://github.com/denoland/deno/pull/32770))

また、`cpSync()`の実装の大部分がRustに移植され、パフォーマンスと互換性が改善されています ([#32687](https://github.com/denoland/deno/pull/32687))

#### `node:events`

`listenerCount()`で`EventTarget`がサポートされています ([#32755](https://github.com/denoland/deno/pull/32755))

#### `node:net`

`Server#listen()`や`connect()`などで`net`パーミッションの有無がチェックされるように挙動が修正されています ([#32721](https://github.com/denoland/deno/pull/32721))

#### `node:dgram`

`Socket#send()`で`net`パーミッションの有無がチェックされるように挙動が修正されています ([#32774](https://github.com/denoland/deno/pull/32774))

また、`Socket#addMembership()`及び`Socket#dropMembership()`の`multicastInterface`引数がサポートされています ([#32754](https://github.com/denoland/deno/pull/32754))

#### `node:http`

`Server`で`CONNECT`メソッドに対する`connect`イベントの発火がサポートされています ([#32599](https://github.com/denoland/deno/pull/32599))

#### `.npmrc`

`certfile`及び`keyfile`がサポートされています ([#32655](https://github.com/denoland/deno/pull/32655))

### Web API

#### `QuotaExceededError`

`QuotaExceededError`クラスが追加されています ([#32244](https://github.com/denoland/deno/pull/32244))。現状ではまだ対応されていなさそうですが、今後、`quota`などのフィールドを参照できるようにする余地がありそうです ([whatwg/webidl#1465](https://github.com/whatwg/webidl/pull/1465))

#### `DOMException`

`DOMException`に対する`structuredClone()`のサポートが追加されています ([#32675](https://github.com/denoland/deno/pull/32675))

#### Console API

Console APIで`%j`がサポートされています ([#32684](https://github.com/denoland/deno/pull/32684))
    
```javascript
console.log("%j", { a: 1 }) // => {"a":1}
```
    
### パフォーマンス改善

複数の[ライフサイクルスクリプト]({{< ref "articles/deno/v1.45.md" >}})を並列で実行するように挙動が改善されています ([#32666](https://github.com/denoland/deno/pull/32666))

また、Linuxにおいて大規模なコードベースのプロジェクトにおけるメモリ使用量を削減するため、各種モジュールの読み込み後や`Worker`の終了時に`malloc_trim(0)`を実行するように修正されています ([#32662](https://github.com/denoland/deno/pull/32662), [#32617](https://github.com/denoland/deno/pull/32617))

その他にも、`URLPattern` ([#32766](https://github.com/denoland/deno/pull/32766))や `TextEncoder`/`TextDecoder` ([#32735](https://github.com/denoland/deno/pull/32735)) の最適化や、`Deno.FsFile.writable`による書き込みにバッファリングを適用することによるパフォーマンス改善 ([#32676](https://github.com/denoland/deno/pull/32676)) も実施されています。

## Deno Deploy

### Deno Deployのアップデート

Deno Deployのchangelogが更新されています:

- [March 12th deployng changelog #2970](https://github.com/denoland/docs/pull/2970)

以下の改善が実施されているようです:

- [非ClassicバージョンのDeno Deploy]({{< ref "articles/2025/10/05.md" >}})における[`Deno.cron()`]({{< ref "articles/deno/v1.38.md" >}})のサポート
- アプリケーションの作成以降のタイミングにおけるデプロイ先リージョンの変更がサポート
- Organizationの削除がサポート
- TanStack Startのサポートの追加
- 外部OpenTelemetryコレクターへの接続がサポート
- ランタイムのメモリ制限の設定がサポート
- 料金プランとして **Builderプラン** が追加

### Deno Deploy Classicの停止について

Deno Deployの旧バージョン ([Deno Deploy Classic]({{< ref "articles/2025/10/05.md" >}}))が2026/07/20に停止されることが計画されているようです:

- [Deno Deploy Classic Sunset #2979](https://github.com/denoland/docs/pull/2979)

2026/07/20までにDeno Deployの新規バージョン (旧称: [Deno Deploy EA]({{< ref "articles/2025/06/01.md" >}})) への移行が推奨されます。

[マイグレーションガイド](https://github.com/denoland/docs/blob/a57b9860203a9895065c038a03a0590c34bd4029/deploy/migration_guide.md)も公開されています。

## `deno lint`コマンドにおける`deno_lint`から`oxlint`への置き換えについて

まだDraft状態ですが、`deno lint`コマンドの内部実装を[`deno_lint` crate](https://github.com/denoland/deno_lint)から[oxlint](https://www.npmjs.com/package/oxlint)へ置き換えるPRが作成されています (**⚠️現時点では、まだマージはされていません。また、正式に導入される計画があるかどうかは現時点ではわかりません。**):

- [feat(lint): replace deno_lint with oxlint subprocess #32852](https://github.com/denoland/deno/pull/32852)

現時点 ([934251b](https://github.com/denoland/deno/pull/32852/changes/934251b553dc0907e59ea2b4661582a82a10ba21))においては、単純に`deno run`コマンドによって`npm:oxlint`を`--format json`付きで実行し、その出力結果を解析する形で実装されています。

また、`deno fmt`コマンドの内部実装を[oxfmt](https://www.npmjs.com/package/oxfmt)へ置き換える対応も上記PR内で実施されています。
