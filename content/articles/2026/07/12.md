## Deno v2.9.2

[Deno v2.9.2](https://github.com/denoland/deno/releases/tag/v2.9.2)がリリースされています。

### `deno desktop`

#### `--hmr` - フレームワークとの統合

[フレームワークの検出機能]({{< ref "articles/deno/v2.8.md" >}})によってフレームワークが検出され かつ `--hmr`オプションが指定されている場合、該当フレームワークのdevサーバーが起動されるように改善されています ([#35722](https://github.com/denoland/deno/pull/35722), [#35851](https://github.com/denoland/deno/pull/35851))。ただし、まだすべてのフレームワークで本機能がサポートされているわけではなく、現状ではFreshと各種Viteベース (Nuxt, Sveltekitなど)のフレームワークのみがサポートされているようです。

#### React Router (フレームワークモード) のサポート

`deno desktop`及び`deno compile`の[フレームワーク検出機能]({{< ref "articles/deno/v2.8.md" >}})においてReact Routerのフレームワークモードがサポートされています ([#35557](https://github.com/denoland/deno/pull/35557))。`@react-router/dev`パッケージの有無に基づいて検出されます。

- **公式ドキュメント**: [docs(desktop): document React Router support (#3361)](https://github.com/denoland/docs/pull/3361)

#### `opacity`/`transparent`

ウィンドウの透明度の調整がサポートされています ([#35646](https://github.com/denoland/deno/pull/35646))。

下記APIによって利用できます。

- `Deno.BrowserWindowOptions`の`opacity`及び`transparent`オプション
- `Deno.BrowserWindow`の`getOpacity()`及び`setOpacity()`メソッド

#### `--exclude-unused-npm`

`deno compile`で提供されている[`--exclude-unused-npm`]({{< ref "articles/2026/06/14.md" >}})でオプションが`deno desktop`コマンドにおいてもサポートされています ([#35740](https://github.com/denoland/deno/pull/35740))

#### `desktop.backend`

`deno.json`における`desktop.backend`フィールドの設定値が適用されない問題が修正されています ([#35815](https://github.com/denoland/deno/pull/35815))

### セキュリティ

#### `minimumDependencyAge`

`deno.json`の`minimumDependencyAge.exclude`においてワイルドカード形式によるパッケージ指定がサポートされています ([#35746](https://github.com/denoland/deno/pull/35746))

```json
{
  "minimumDependencyAge": {
    "age": 1440,
    "exclude": [
      "jsr:@std/*"
    ]
  }
}
```

#### `node:net`

[Deno v2.9]({{< ref "articles/deno/v2.9.md" >}})における対応に続き、`node:net`によってUNIXソケットへ接続またはlistenする際も`--allow-net=unix:<path>`が要求されるように挙動が変更されています ([#35835](https://github.com/denoland/deno/pull/35835))

#### `--ignore-read`

[`--ignore-read`]({{< ref "articles/deno/v2.6.md" >}})オプションにコンマ区切りで複数のパス指定がサポートされています ([#35661](https://github.com/denoland/deno/pull/35661))

### OpenTelemetry

下記環境変数がサポートされています ([#35068](https://github.com/denoland/deno/pull/35068))

- [`OTEL_SPAN_ATTRIBUTE_VALUE_LENGTH_LIMIT`](https://github.com/open-telemetry/opentelemetry-specification/blob/a824fb4eba795c5c65dd397b3d22e7c28e934de3/specification/configuration/sdk-environment-variables.md#span-limits)
- [`OTEL_ATTRIBUTE_VALUE_LENGTH_LIMIT`](https://github.com/open-telemetry/opentelemetry-specification/blob/a824fb4eba795c5c65dd397b3d22e7c28e934de3/specification/configuration/sdk-environment-variables.md#attribute-limits)

_Attribute_ に長い文字列が設定された場合、これらの環境変数で指定された長さに収まるように切り捨てられます。

### `--inspect`

Node.jsと同様に、`SIGUSR1`シグナルを受信した際にインスペクター (`--inspect`) が起動されるように改善されています ([#35738](https://github.com/denoland/deno/pull/35738))

- **公式ドキュメント**: [document SIGUSR1 inspector activation (#3388)](https://github.com/denoland/docs/pull/3388)

### `node:v8`

`setHeapSnapshotNearHeapLimit()`が実装されています ([#35694](https://github.com/denoland/deno/pull/35694))

利用するためにはカレントディレクトリに対する書き込み権限 (`--allow-write`) が必要です。

### `node:test`

`getTestContext()`が実装されています ([#35678](https://github.com/denoland/deno/pull/35678))

### `node:child_process`

`createWritestream()` (`node:fs`) や`Socket` (`node:net`) などの`fd`オプションに対して、`spawn()`などの`stdio`オプションで設定された4つ目以降の追加の _fd_ を指定できるように改善されています ([#34133](https://github.com/denoland/deno/pull/34133), [#35805](https://github.com/denoland/deno/pull/35805))

### `node:fs`

`glob()`の`followSymlinks`オプションがサポートされています ([#35524](https://github.com/denoland/deno/pull/35524))

### Web Cache API

`Cache#keys()`が実装されています ([#35455](https://github.com/denoland/deno/pull/35455))

### パフォーマンス改善

#### `deno install`

npmパッケージに関するメタデータの読み込みを最適化することで、インストールが高速化されています ([#35698](https://github.com/denoland/deno/pull/35698))。ベンチマークによるとDrizzle ORMのインストールが2倍以上高速化されているようです。

#### Streams API

Streams APIのパフォーマンス最適化が実施されています ([#35689](https://github.com/denoland/deno/pull/35689), [#35773](https://github.com/denoland/deno/pull/35773), [#35788](https://github.com/denoland/deno/pull/35788), [#35790](https://github.com/denoland/deno/pull/35790), [#35799](https://github.com/denoland/deno/pull/35799), [#35810](https://github.com/denoland/deno/pull/35810), [#35813](https://github.com/denoland/deno/pull/35813), [#35842](https://github.com/denoland/deno/pull/35842), [#35844](https://github.com/denoland/deno/pull/35844))

主要な改善として、`pipeTo()`などによって接続された`ReadableStream`及び`WritableStream`がどちらもDenoによって管理されるリソース (例: `Deno.FsFile.readable`, `Deno.stdout.writable`, など) 由来のものであった場合に、Rust ↔ JavaScript 間でのデータのコピーを抑制することによるオーバーヘッドの削減が行われているようです ([Fast streams (#16046)](https://github.com/denoland/deno/issues/16046))。特定のファイルの内容を`/dev/null`に書き込むことで`pipeTo()`のパフォーマンスを計測するベンチマークによると、 7.5倍ほどの高速化が確認されているようです ([#35689](https://github.com/denoland/deno/pull/35689))。

### `queueMicrotask()`

`queueMicrotask()`が独自の実装ではなくV8から提供される実装 (`--enable-queue-microtask`) を利用するように変更されています ([#35787](https://github.com/denoland/deno/pull/35787))。パフォーマンスの改善が見込まれるようです。
